Introduction

Ce manuel décrit les fonctionnalités de l’extension MyParcel Belgique pour Magento 2.

Cette extension Magento 2 vous permet de créer deux types d’envois MyParcel Belgique :

  1. Normal (en Belgique, avec de nombreuses options de livraison et de moment d’envoi)
  2. colis vers l’étranger, uniquement vers l’UE.

Sommaire

1 Installation et mise à jour

1.1 Exigences techniques

Pour utiliser l'extension MyParcel Belgique Magento 2, votre serveur doit répondre à l’une des conditions suivantes:

  • Version de PHP 5.6
  • Version de PHP 7 ou supérieure

La version de Magento suivante est supportée:

  • Community 2.x

1.2 Installation

L’extension MyParcel Belgique Magento 2 ne peut pas être téléchargée via le Magento Marketplace. 

Nous vous recommandons de faire une sauvegarde de votre environnement Magento 2 avant toute installation. L’extension est disponible via le terminal. Consultez votre administrateur web. Votre administrateur web peut exécuter la commande suivante à la racine du site (par ex. public_html) pour installer l’extension MyParcel Belgique: 
composer require myparcelbe/magento 

Vous installez une extension dans votre environnement Magento pour la première fois?
Dan bestaat de kans dat je de volgende melding krijgt:



Om de Username en Password te verkrijgen ga je naar https://marketplace.magento.com. Hier kun je een account aanmaken en inloggen. Ga daarna in My Account naar Marketplace. Klik vervolgens op My Access Keys en op Magento 2.

Hier zie je public key en een private key. De public key kan je in de terminal gebruiken als username en de private key kan je gebruiken als Password. Zie je de keys nog niet? Klik dan op Create A New Access Key.



Pour vider le cache et exécuter d’autres actions nécessaires de Magento 2, utilisez la commande suivante: rm -rf var/di/*; rm -rf pub/static/*; rm -rf var/cache/*; rm -rf var/pagecache/*; rm -rf var/generation/*; rm -rf var/dir/*; rm -rf var/view_preprocessed/*; php bin/magento cache:flush && php bin/magento setup:upgrade && php bin/magento setup:static-content:deploy && php bin/magento setup:static-content:deploy nl_NL && php bin/magento indexer:reindex

1.3 Mise à jour

Pour effectuer une mise à jour, utilisez la commande suivante: 
composer update myparcelbe/magento

Si vous utilisez actuellement la version 1.x de MyParcel Belgique et si vous souhaitez la mettre à jour avec la version 2.x, vous devrez utiliser la commande suivante pour installer la dernière version: 
composer require myparcelbe/magento "~2.1"

Pour vider le cache et exécuter d’autres actions nécessaires de Magento 2, utilisez la commande suivante: rm -rf var/di/*; rm -rf pub/static/*; rm -rf var/cache/*; rm -rf var/pagecache/*; rm -rf var/generation/*; rm -rf var/dir/*; rm -rf var/view_preprocessed/*; php bin/magento cache:flush && php bin/magento setup:upgrade && php bin/magento setup:static-content:deploy && php bin/magento setup:static-content:deploy nl_NL && php bin/magento indexer:reindex

2 Configuration

Allez dans le menu Stores → Configuration → MyParcel pour configurer l’extension MyParcel Belgique.


Vous aurez également le choix entre 4 onglets. Dans l’onglet ‘version’, vous pouvez vérifier la version du plug-in que vous avez installée. Dans l’onglet ‘Paramètres généraux’, vous pourrez introduire votre clé API et définir vos préférences d'impression. L’onglet ‘Paramètres d’expédition standards’ vous donne la possibilité de définir vos préférences d’options d’expédition standards de MyParcel Belgique. L’onglet ‘Paramètres de checkout’ vous permet de configurer des options d'expédition complémentaires que vos clients pourront choisir dans le panier.

2.1 Paramètres généraux

2.1.1 Paramètres de la clé API

Vous pouvez introduire ici votre clé API. Celle-ci est nécessaire au transfert des envois vers MyParcel. Ces données peuvent être retrouvées dans votre compte MyParcel Belgique sur backoffice.sendmyparcel.be en choisissant Paramètres -> Général.

2.1.2 Orientation de l’impression

Sélectionnez orientation d’impression. Si vous choisissez le format A4, quatre étiquettes d'expédition seront imprimées par feuille A4. Si vous choisissez le format A6, vous aurez besoin d’étiquettes d’expédition qui sont compatibles avec une imprimante à étiquettes spéciales comme une Zebra ou une Dymo 4XL. 

2.2 Paramètres d'expédition standards

Dans Paramètres d’expédition standards vous pouvez configurer le mode de traitement par défaut de votre envoi lorsque vous utilisez l'extension MyParcel Belgique pour Magento 2.

2.2.1 Options d'expédition standards

Les options suivantes sont activées en fonction de la valeur de la commande. Exemple : si vous saisissez la valeur 100 dans Assurance jusqu’à 500 € les commandes d'une valeur d’achat supérieure à 100 € seront automatiquement assurées à concurrence de 500 €.

  • Signature pour réception
  • Assurance jusqu'à 500 €

Attention : l’option n’est pas activée si Non est complété. L’option est active si la valeur Oui est saisie. L’option est également activée lorsque ‘0’ est saisi dans ‘A partir du prix’.

3 Fonctionnalités back-end

3.1 Créer un envoi

Vous pouvez créer un envoi MyParcel Belgique de deux manières:

1.Méthode par lots via l’aperçu des commandes.

  1. Sélectionnez les commandes que vous voulez exporter vers MyParcel Belgique.




  2. Dans la barre d’actions, sélectionnez Imprimer les étiquettes MyParcel Belgique.



  3. Sélectionnez la position d’impression correcte si vous avez choisi ‘A4’.


  4. Le PDF apparaît sur votre ordinateur et est prêt à être imprimé.

  5. WSi vous souhaitez ignorer les paramètres de l’étape 3, parce que vous avez par exemple déjà défini les options d'expédition correctes pour vos commandes via les ‘Paramètres d'expédition standards’, vous pouvez alors télécharger immédiatement vos étiquettes via ‘Imprimer directement les étiquettes MyParcel’.



2. Par commande, lorsque vous créez un envoi Magento.

  1. Cliquez sur la commande correspondante et choisissez le bouton Envoyerdans le coin supérieur droit.


  2. Un cadre apparaît dans le coin inférieur droit. Vous pouvez ici adapter le type d’envoi, de même que les options d’expédition disponibles par type d’envoi.




  3. Envoyez la commande vers MyParcel Belgique via Enregistrer l’envoi.Cela permet la création d’un concept d'envoi.

  4. Vous pourrez ensuite immédiatement imprimer l’étiquette dans la commande, en cliquant sur le bouton Imprimer l'étiquette en haut à droite.



3.2 Aperçu des commandes

Afin de vous fournir à tout moment une vision claire sur le statut de vos envois, 3 colonnes sont automatiquement ajoutées à l’aperçu des commandes après l'installation du plug-in MyParcel Belgique.



  • Suivi du statut: vous pouvez voir ici le statut de l’envoi, constamment mis à jour.

  • Numéro de suivi: vous trouverez ici le code-barres de votre envoi.

  • Action de suivi: lorsque la commande n’a pas encore été envoyée et que vous cliquez sur ‘Select’, une fenêtre contextuelle contenant des options de téléchargement des étiquettes apparaît. Si votre commande a déjà été expédiée, vous pourrez télécharger l’étiquette à nouveau.


3.3 Modèle d’e-mail Track & Trace

Lorsqu’un e-mail de confirmation de commande est envoyé au client depuis Magento, le code-barres de l’envoi est repris automatiquement. Cela ne doit donc pas être configuré séparément. Si vous souhaitez toutefois envoyer un autre e-mail Track & Trace au client à partir de Magento 2, allez dans Boutiques → Configuration → Vente → E-mail de vente → Codes-barres MyParcel, et définissez votre propre modèle.

4 Checkout instellingen

Sous l’onglet ‘Paramètres de checkout’, vous pouvez appliquer tous les paramètres qui seront visibles au moment du checkout. Cet onglet est subdivisé en ‘Paramètres de base’, ‘Livraison’, ‘Petit paquet pour boîte aux lettres’, ‘Livraison le matin’, ‘Livraison en soirée’, ‘Collecte’ et ‘Collecte anticipée’.

4.1 Paramètres de checkout

4.1.1 Dernier moment de commande

Vous pouvez définir ici jusqu’à quel moment il vous est possible de préparer une commande pour la livraison. Si vous saisissez 16:00, la première possibilité de livraison sera portée à un jour ultérieur pour tous les clients qui passeront commande après cette heure.
Par exemple : votre délai de traitement est d’un jour et vous pouvez traiter les commandes jusqu’à 16h00. Pour une commande reçue le mardi 4 octobre à 16h02, la première possibilité de livraison pour votre client sera donc le jeudi 6 octobre. Cela signifie que vous devrez déposer votre colis le mercredi à un point bpost.



4.1.2 Jours d’envoi

Dans ‘jours d'expédition’, vous pouvez indiquer les jours lors desquels il vous est possible de fournir des colis pour un enlèvement le jour même. Au niveau du checkout, cet élément sera pris en compte pour les options proposées à votre client.

4.1.3 Retard

Vous pouvez indiquer ici si vous avez besoin d’un délai de traitement plus long pour procéder à la livraison. Au niveau du checkout, cet élément sera pris en compte pour les jours de livraison possibles proposés à votre client.

4.2 Livraison

4.2.1 Samedi

Vous pouvez également choisir de faire livrer un colis le lundi. N’oubliez pas de sélectionner le samedi comme jour d’envoi.
Attention : la livraison le samedi n’est possible que lorsque le dernier moment de commande est fixé au vendredi 14:30 et que l’envoi est déposé dans l’un des points bpost destinés à cet effet.

4.2.2 Signature pour réception

Vous pouvez indiquer si vos clients peuvent choisir cette option lors du checkout. Si vous souhaitez l’afficher, choisissezOui Il est possible de renommer cette option ; dans l’exemple, elle s’appelle ‘Signature pour réception’. Vous pouvez également associer des frais complémentaires à cette option. Dans l'exemple cette option coûte 0,36 € supplémentaires.

Une option a été également créée pour un supplément combiné des choixLivrer uniquement chez moi et Signature pour réception. Vous pouvez remplir un prix à votre discrétion.

4.3 Collecte dans un point bpost

Avec cette option, vos clients peuvent choisir d’aller chercher leur colis dans un point bpost. Ils peuvent choisir le point en question lors du checkout. Vous pouvez éventuellement facturer un supplément pour cela.

5 Questions fréquemment posées

5.1 Mon système de checkout est-il supporté?

Les checkouts suivants ont été testés et sont pris en charge:

  • OnePageCheckout, qui est le checkout standard de Magento.
  • MagePlaza One Step Checkout

5.2 J’utilise une extension de vérification du code postal, est-ce un problème?

Il est possible d’utiliser une extension de vérification de code postal en combinaison avec l’extension MyParcel Belgique Magento 2. Le lien suivant vous amène sur le site proposant l’extension que nous vous recommandons. 

Postcode Checkout Magento Extensie

5.3 Comment puis-je configurer la livraison gratuite?

A côté des Tables Rates, vous pouvez également définir un envoi gratuit via Marketing > Règles de prix du panier

  1. Cliquez en haut à droite sur Ajouter une nouvelle règle.
  2. Donnez un nom à la règle dans Données de la règle (par ex. : ‘Envoi gratuit à partir de 50’). Activez le statut et sélectionnez toutes les options dans sites internet et groupes de clients.
  3. En haut à gauche, cliquez sur Conditions et précisez vous-même la règle en cliquant sur +. (Exemple : ‘Sous-total est égal ou supérieur à 50’).
  4. A l’étape suivante, indiquez ‘Seulement pour les articles concernés’ dans ‘Expédition gratuite’.
  5. Cliquez sur ‘Enregistrer’ dans le coin supérieur droit.
